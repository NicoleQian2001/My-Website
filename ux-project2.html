The product

This website is designed for FindASchool company to offer their service in helping users to browse and find a desired elementary or secondary school. It provides accessibility tools to support users with disabilities.

Responsibilities

Creating paper and digital wireframes, low and high-fidelity prototypes, conducting usability studies, accounting for accessibility, and iterating on designs.

The problem

Parents want to find a school that is best for their children or students want to find a school that spark their interests out of a number of school, but website of each school is structured differently. It’s time-consuming to gather a set of desired indicators of each school.

The goal

Design a responsive website that allows users to find and compare a set of schools’ information in a fast and easy way without accessibility issues.

User research: summary

I conducted interviews and created empathy maps to understand the users I’m designing for and their needs. A primary user group identified through research was busy parents selecting an elementary and/or secondary school out of all schools for their children. This user group confirmed initial assumptions about the service’s customers, but research also revealed that excessive time spent gathering the same set of information from each school’s website was not the only users pain point. Other user problems included accessibility issues (visual impairment, language barrier, etc) and an incomplete list of schools to select from.

Time

Working parents are too busy to browse through each school website to gather the same set of information

Accessibility

Many school websites are not equipped with assistive technologies.

IA

Text-heavy websites are often difficult to read and extract useful information from

Paper wireframes

Taking the time to draft iterations of each screen of the app on paper ensured that the elements that make it to digital wireframes would be well-suited to address user pain points. For the homepage, I made several horizontal scrollable sections, each indicating a type of school with the best rankings, so users can browse schools that might capture their interests visually and efficiently.

For the home screen, I prioritized a quick and easy ordering process to help users save time which has categories and snacks selection.

Accessibility Considerations

The “Read this page” function next to the search bar is a screen reader that helps visually impaired users to obtain information on the page.

The language menu next to the search bar supports changing the website’s text to a different language. This allows users to browse the website without language barriers.

Increasing the font size next to the search bar helps users with poor vision see the text better.

Takeaways

Impact: A study participant stated that “I would definitely use this to find a school. It’s so easy and fast to find the same set of indicators and information for all schools. All the information I look for is presented too.”

What I learned: While designing the responsive website, I learned that the first ideas for the app are only the beginning of the process. Usability studies and peer feedback influenced each iteration of the app’s designs.
